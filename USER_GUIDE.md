# EduSync User Guide

This guide explains how each type of EduSync user can access the application, navigate their module, view data, and complete their daily tasks. EduSync has three main user roles:

- Admin
- Teacher
- Student

Each role sees a different set of pages based on their permissions. The application protects each module so users can only access the pages that belong to their role.

## 1. Getting Started

Open the EduSync application in the browser. The first screen is the login page.

On the login page, choose your role:

- Admin
- Teacher
- Student

Then enter:

- Email address.
- Password.

Select **Sign in** to enter the application.

After login, EduSync automatically sends each user to the correct dashboard:

| User Type | First Page After Login |
| --- | --- |
| Admin | Admin Dashboard |
| Teacher | Teacher Dashboard |
| Student | Student Dashboard |

If a user logs out, EduSync clears the saved session and returns to the login page.

## 2. Common Layout for All Users

All modules use a similar layout so the application feels consistent.

### Sidebar

The sidebar appears on the left side of the screen. It contains the pages available to the logged-in user.

The sidebar is divided into sections:

- Overview
- Management
- Operations

At the bottom of the sidebar, users can see:

- Logout button.
- User initials.
- User name.
- User role.

### Main Page Area

The main area displays the selected page. This is where dashboards, lists, forms, reports, and records appear.

### Header Action Bar

Many pages include an action bar with:

- Edit
- Refresh
- Delete
- Export

Some pages are read-only or require selecting a record first. In those cases, EduSync shows a helpful popup instead of performing the action directly.

### Search, Filters, and Columns

List pages commonly include:

- Search box.
- Filter button or dropdown.
- Configure columns button.
- Pagination controls.

Users can search records, filter lists, and choose which columns are visible.

## 3. Admin User Guide

The Admin module is used by school administrators. Admin users can manage the complete school system.

### What Admin Users Can Access

Admin users can access:

- Dashboard
- Announcements
- Students
- Teachers
- Classes
- Subjects
- Attendance
- Exams
- Fee Management
- Reports

### Admin Dashboard

The Admin Dashboard gives a full overview of the school.

Admins can see:

- Total students.
- Total teachers.
- Attendance today.
- Fee collection.
- Attendance by grade.
- Upcoming events.
- Recent enrollments.

Use this page to quickly understand the current school status.

### Announcements

Admins use Announcements to create and manage school notices.

Admins can view information such as:

- Announcement title.
- Announcement type.
- Message details.
- Date.
- Target audience.

Use this page to communicate important updates, such as:

- Exam reminders.
- Fee deadlines.
- Parent-teacher meetings.
- School events.

### Students

The Students page allows admins to manage student records.

Admins can see:

- Student name.
- Phone number.
- Fee status.
- Section.
- Status.
- Grade.
- Student email.
- Admission date.

Admins can use this page to:

- Add a new student.
- Search for a student.
- View student records.
- Configure list columns.
- Export student records.

#### Adding a Student

Select **Add student** to open the student form.

The student form is divided into steps:

1. Personal information.
2. Academic details.
3. Guardian information.
4. Fee and documents.

Fill in the required fields and move through the steps. The stepper shows where the user is in the process.

### Teachers

The Teachers page allows admins to manage staff records.

Admins can see and manage:

- Teacher name.
- Gender.
- Phone number.
- Work email.
- Designation.
- Employment type.
- Joining date.
- Professional details.

#### Adding a Teacher

Select **Add teacher** to open the teacher form.

The form includes:

- Personal information.
- Professional details.
- Employee ID.
- Designation.
- Employment type.
- Work email.
- Joining date.

Select **Add teacher** to save the teacher or **Cancel** to close the form.

### Classes

The Classes page is used to create and manage classes.

Admins can see:

- Grade.
- Section.
- Maximum capacity.
- Classroom number.
- Academic year.
- Subjects.
- Teacher assignment.
- Status.

#### Adding a Class

Select **Add class** to open the class form.

Enter:

- Grade.
- Section.
- Maximum capacity.
- Classroom number.
- Academic year.
- Optional class notes.

Select **Create class** to save.

### Subjects

The Subjects page is used to manage curriculum subjects.

Admins can see:

- Subject name.
- Subject code.
- Grade level.
- Subject category.
- Lab requirement.
- Elective setting.

#### Adding a Subject

Select **Add subject** to open the subject form.

Fill in:

- Subject name.
- Subject code.
- Grade level.
- Category.
- Settings such as elective subject and lab requirement.

Select **Add Subject** to save.

### Attendance

The Attendance page allows admins to view and manage attendance records.

Admins can see:

- Student attendance status.
- Attendance by date.
- Attendance history.
- Expandable attendance records by student.

Admins can use attendance tools to:

- Mark attendance.
- Review student attendance.
- Check present, absent, late, and holiday records.

### Exams

The Exams page is used to manage exam schedules.

Admins can see:

- Exam title.
- Subject.
- Class.
- Date.
- Marks.
- Status.

Admins can schedule and manage exam information from this page.

### Fee Management

The Fee Management page tracks student payments.

Admins can see:

- Fee records.
- Student information.
- Fee month.
- Amount received.
- Payment method.
- Transaction ID.
- Payment status.

#### Recording a Fee Payment

Select the payment action to open the Record Fee Payment form.

The form includes:

- Student search.
- Fee month.
- Amount received.
- Payment method.
- Transaction ID.

Select **Save Payment** to record the payment.

### Reports

The Reports page allows admins to generate and view school reports.

Admins can generate reports for:

- Attendance.
- Academic records.
- Fees.
- General school data.

Reports can be exported or downloaded depending on the available format.

## 4. Teacher User Guide

The Teacher module is designed for teachers to manage their assigned academic responsibilities.

### What Teacher Users Can Access

Teacher users can access:

- Dashboard
- Announcements
- Students
- Classes
- Subjects
- Attendance
- Exams
- Reports

Teachers do not see admin-only management pages such as the full teacher staff management page or admin fee management.

### Teacher Dashboard

The Teacher Dashboard gives a quick view of the teacher's assigned work.

Teachers can see:

- Assigned classes.
- My students.
- Present today.
- Upcoming exams.
- My Classes panel.
- Upcoming Exams panel.

Teachers can use:

- Notification button to view dashboard alerts.
- Profile initials button to view teacher summary.
- View all links to open full Classes or Exams pages.
- Header export button to download dashboard metrics.

### Announcements

Teachers can view school announcements relevant to them.

Teachers can:

- Search announcements.
- Filter by announcement type.
- View announcement details.

The New Announcement button shows a message that announcement creation is handled by the Admin module.

### Students

The Teacher Students page shows students connected to the teacher's assigned classes.

Teachers can see:

- Student name.
- Roll number.
- Class.
- Guardian.
- Fee status.
- Status.
- Email.
- Phone number.

Teachers can:

- Search by student name or roll number.
- Filter by assigned class.
- Configure visible columns.
- Export student data.

This helps teachers quickly look up students in their own classes.

### Classes

The Teacher Classes page shows classes assigned to the teacher.

Teachers can see:

- Class name.
- Room.
- Academic year.
- Subjects.
- Schedule.
- Status.
- Capacity.
- Teacher assignment.

Teachers can search by:

- Grade.
- Section.
- Room number.

This page helps teachers review their class responsibilities.

### Subjects

The Teacher Subjects page shows curriculum subjects connected to the teacher or assigned grades.

Teachers can see:

- Subject name.
- Subject code.
- Grade.
- Category.
- Weekly periods.
- Lab status.
- Teacher assignment.

Teachers can:

- Search subjects.
- Filter by category.
- Configure columns.
- Export subject data.

### Attendance

The Teacher Attendance page is used to mark attendance for assigned students.

Teachers can:

- Choose date.
- Choose class.
- Search students.
- Mark all present.
- Mark all absent.
- Mark individual students as Present, Absent, Late, or Holiday.
- Save attendance.

#### Marking Attendance

1. Open the Attendance page.
2. Select the attendance date.
3. Select a class or keep All Classes.
4. Use Mark all Present/Absent if needed.
5. Adjust individual student statuses if required.
6. Select **Save attendance**.

If no student is marked, EduSync shows a popup asking the teacher to mark at least one student.

### Exams

The Teacher Exams page shows exam records and allows teachers to schedule exams.

Teachers can see:

- Exam title.
- Subject.
- Class.
- Date.
- Marks.
- Status.

Teachers can search exams and filter by class.

#### Scheduling an Exam

Select **Schedule exam** to open the exam form.

Fill in:

- Exam title.
- Exam type.
- Subject.
- Class and section.
- Exam date.
- Start time.
- Duration.
- Total marks.
- Passing marks.
- Negative marking.
- Optional instructions.
- Publish exam to students setting.

Select **Schedule Exam** to save.

If required fields are missing, EduSync shows a validation popup.

### Reports

The Teacher Reports page shows teacher-related report exports.

Teachers can:

- Generate a report.
- Select report cards.
- Download a report.
- Export selected reports.
- Clear selected reports.

If no report is selected before export, EduSync shows a helpful popup.

## 6. How Users Access Data

EduSync uses the logged-in user's role to decide which pages and records are available.

### Admin Data Access

Admins have the broadest access. They can view and manage school-wide records:

- All students.
- All teachers.
- All classes.
- All subjects.
- Attendance records.
- Exams.
- Fee payments.
- Reports.
- Announcements.

### Teacher Data Access

Teachers access data related to their teaching responsibilities:

- Assigned classes.
- Students in assigned classes.
- Subjects connected to assigned grades or teacher assignment.
- Attendance for assigned students.
- Exams for assigned classes.
- Teacher reports.
- Teacher announcements.

### Student Data Access

Students access personal and grade-related data:

- Own profile.
- Own attendance.
- Own fee records.
- Own exam schedule.
- Own reports.
- Subjects for their grade.
- Announcements visible to students.

## 7. Common Actions Explained

### Searching Records

Use the search box at the top of list pages. Search usually supports names, roll numbers, class names, subjects, or report names depending on the page.

### Filtering Records

Some pages include filters such as:

- Class filter.
- Category filter.
- Announcement type filter.

Use these controls to narrow the list.

### Configuring Columns

On list pages, select **Configure columns** to choose which columns are visible. This helps users focus on the data they need.

### Exporting Data

Select **Export** in the header to download available records. Pages that support export create a CSV file.

### Refreshing Data

Select **Refresh** to reload or refetch page data.

### Popups and Confirmations

EduSync uses styled popups for:

- Warnings.
- Save confirmations.
- Validation messages.
- Detail previews.
- Read-only explanations.

These popups guide users without leaving the current page.

## 8. Recommended Daily Use

### Admin Daily Checklist

1. Open the Admin Dashboard.
2. Review attendance, fee collection, and recent activity.
3. Check announcements and events.
4. Manage new student or teacher records if needed.
5. Review attendance, exams, fees, and reports.
6. Export required records.
7. Log out.

### Teacher Daily Checklist

1. Open the Teacher Dashboard.
2. Review assigned classes and upcoming exams.
3. Check announcements.
4. Open Attendance and mark today's attendance.
5. Review assigned students and classes.
6. Schedule exams if needed.
7. Download reports if required.
8. Log out.

### Student Daily Checklist

1. Open the Student Dashboard.
2. Read new announcements.
3. Check subjects and upcoming exams.
4. Review attendance history.
5. Check fee status.
6. Open reports if needed.
7. Log out.

## 9. Troubleshooting

### User Cannot Log In

Check:

- Correct role is selected.
- Email is correct.
- Password is correct.
- Backend server is running.

### User Opens Wrong Page

EduSync automatically redirects users away from pages they are not allowed to access.

### Data Does Not Appear

Try:

- Refreshing the page.
- Checking search/filter values.
- Confirming the backend API is running.
- Logging out and logging back in.

### Export Has No Data

If there are no records on the page, EduSync shows a popup explaining that there is nothing to export.

## 10. Summary

EduSync provides a complete school management experience for three user groups:

- Admins manage the full school system.
- Teachers manage assigned classroom workflows.
- Students view their own academic and fee information.

Each module is role-specific, easy to navigate, and built around consistent pages, tables, popups, and actions.
